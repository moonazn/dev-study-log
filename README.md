# 📑 Dev Study Log

이 저장소는 이전에 공부했던 내용을 다시 꺼내 보면서 더 깊게 이해하는 나의 학습 과정을 담기 위해 만든 곳이다.
<br></br>

단순한 문법과 개념, 사용법을 정리하기보다
> “왜 이렇게 동작할까?”
> 
> “컴파일러는 내부적으로 어떻게 처리할까?”
> 
> “이 설계는 왜 이렇게 되어 있을까?”

와 같은 질문을 스스로 계속 던지고 그 질문을 따라가며 공부한다.
<br></br>

나는 무엇이든 **아는 만큼 보인다고 생각한다.**


프로그램이 왜 그렇게 동작하는지 이해해야 더 좋은 설계를 고민할 수 있고, 이는 더 나은 사용자 경험을 만들기 위한 선택의 과정이다.


앞으로 어떤 기술을 배우더라도 ‘사용’에 그치지 않고 **원리를 이해하고 주체적으로 선택할 수 있는 개발자**가 되는 것이 목표이다.

<br></br>
### 🐟 공부 방식에 따른 폴더링

이 저장소에는 **공부 결과**보다 **공부 과정**을 기록한다.

하나의 개념을 공부하다가 생긴 질문이 또 다른 질문으로 이어진다.

이렇게 파고드는 과정에서 개념들이 자연스럽게 연결된다.
<br></br>

따라서 폴더는 개념에 따른 분류가 아닌 **공부를 시작한 주제**를 기준으로 구성한다.

예를 들어 Swift 폴더 안의 문서에 운영체제나 자료구조와 관련된 내용이 함께 등장할 수 있다.
<br></br>

나에게 중요한 것은 개념을 분리해서 정리한 백과사전 만들기가 아니라 **어떤 질문에서 시작해 어디까지 탐색했는지**이기 때문이다.

<br></br>
